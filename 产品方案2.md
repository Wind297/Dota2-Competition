Dota2 个人积分赛管理系统 - 产品方案与开发逻辑 v1.1
1. 业务背景
5v5 个人积分赛，每场选10人进入游戏随机分队，胜方每人+1分。希望用系统取代手工记录和筛选，提高效率。

2. 核心功能
选手池管理：选手库、在线/离线标记（每个比赛日自动重置为在线）。

多维筛选器：按积分档位、今日状态、在线状态自由组合筛选。

快捷方案：一键筛选。

比赛生命周期：创建比赛 → 确认10人名单 → 录入5名胜者 → 自动计分。

积分排行榜：实时总分排名。

3. 业务规则（重要）
比赛日定义

一个比赛日从 每日早上 9:00:00 开始，到次日 8:59:59 结束。

比赛归属于其创建时间所在的比赛日。

积分与档位

积分 = 累计胜场数，整数。

档位（左闭右开）：

低分档：0～4分

中分档：5～9分

高分档：≥10分（15分以上仍属本档）

当日状态定义

“今日未参赛”：今日参赛场数 = 0

“今日未获胜”：今日胜场数 = 0

“今日参赛≤1场”：今日参赛场数 ≤ 1

在线状态

每个比赛日早上 9:00，系统将所有选手的 is_online 重置为 true。

管理员可手动切换，标记当天不会来的人，或让临时上线的人恢复在线。

快捷筛选方案（匹配修正后档位）

方案名	条件
高分场	积分 ≥10 且在线
中分场	积分 5～9 且今日未参赛 且在线
低分场	积分 0～4 且今日未参赛 且在线
补位场	今日未参赛 且在线
红利场	今日参赛≤1场 且今日未获胜 且在线
加赛场	今日仅参赛1场 且在线
附加赛场	仅在线
数量与并发约束

确认名单必须恰好 10 人；提交结果必须恰好 5 名胜者。

同一比赛日内，一名选手不能同时处于两场 confirmed（已确认未完成）的比赛中。

比赛状态仅 confirmed → completed，不支持界面上的编辑或取消。

4. 数据模型
players

字段	类型	说明
id	int PK	
name	varchar	
current_score	int	累计胜场，默认0
is_online	boolean	当日在线，默认true，9点全局重置
created_at / updated_at	timestamp	
matches

字段	类型	说明
id	int PK	
matchday_start	timestamp	比赛日窗口起点（当日9:00）
actual_time	timestamp	实际开赛时间，可空
status	enum('confirmed','completed')	
created_at	timestamp	
match_players

字段	类型	说明
id	int PK	
match_id	FK→matches	
player_id	FK→players	
is_winner	boolean	赛前NULL，赛后true/false
5. API 大纲（Python 实现）
GET /api/players - 选手列表（含当日统计、在线状态），支持筛选参数

PATCH /api/players/{id} - 切换在线状态

POST /api/matches - 创建比赛，传入 10 个 player_id

PATCH /api/matches/{id}/result - 提交结果，传入 5 个获胜 player_id

GET /api/matches - 历史比赛

GET /api/rankings - 积分排行
