# 开发与服务器更新指南（写给「只会写代码」的自己）

本文假设你：**会在 Windows 上用 Cursor 改代码**，但 **不熟悉 Linux、SSH、Git、Docker**。目标只有两件事：

1. **本机**：把改好的代码 **提交并推送到 GitHub**。  
2. **服务器**（例如已买好的腾讯云轻量）：让上面的网站 **跑新版本**。

先把三个环境分清：

| 环境 | 是什么 | 你平时在哪操作 |
|------|--------|----------------|
| **本机** | 你自己的 Windows 电脑，Cursor 里的项目文件夹 | PowerShell、Cursor |
| **GitHub** | 网上的代码仓库，存 `git push` 上去的历史 | 浏览器打开 github.com |
| **服务器** | 腾讯云上的 Linux 虚拟机，网站真正运行的地方 | 用 SSH「远程登录」后在黑色终端里敲命令 |

代码流向通常是：**本机 →（push）→ GitHub →（pull 或拷贝）→ 服务器 →（build）→ Docker 容器**。

---

## 一、本机：第一次准备 Git（若已能 push 可跳过）

### 1.1 安装 Git

官网下载安装：**https://git-scm.com/download/win**  
安装时一路下一步即可。装好后 **关掉再打开** PowerShell。

### 1.2 确认项目在 Git 管理下

在 PowerShell 里：

```powershell
cd "C:\Users\Jenson.Wang\OneDrive\PythonProject\Cursor\Dota2-Competition"
git status
```

- 若显示 **`On branch main`** 和一堆文件：说明已是仓库，继续下一节。  
- 若提示 **not a git repository**：需要先在 GitHub 建空仓库，再按 GitHub 页面提示 `git init` / `git remote add`（你当前项目应已配置好，一般不必重做）。

### 1.3 配置你的名字和邮箱（仅一次）

```powershell
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

## 二、本机：日常「改代码 → 推到 GitHub」

每次改完功能，在 **项目根目录** `Dota2-Competition` 打开 PowerShell，按顺序执行（**一行一回车**）。

### 2.1 看改了哪些文件

```powershell
cd "C:\Users\Jenson.Wang\OneDrive\PythonProject\Cursor\Dota2-Competition"
git status
```

红色 = 未暂存，绿色 = 已暂存待提交。

### 2.2 把改动放进「待提交区」

```powershell
git add .
```

（若只想提交部分文件，可改成 `git add README.md` 这种具体路径。）

### 2.3 写一句说明并提交到本机 Git

```powershell
git commit -m "简要说明这次改了什么"
```

若提示 **nothing to commit**：说明没有新变化，或都被忽略了，用 `git status` 再看一眼。

**第一次在本机 commit** 若提示要设置身份，按 **一、1.3** 配置后再执行 `git commit`。

### 2.4 推送到 GitHub

```powershell
git push origin main
```

- **`main`** 是分支名；若你仓库默认是 **`master`**，把命令里的 `main` 改成 `master`。  
- 若提示要登录：GitHub **已不支持用网页登录密码** 作为 HTTPS 密码，要在 GitHub 生成 **Personal Access Token（PAT）**，在密码提示处 **粘贴整段 token**（输入时往往不显示字符，属正常）。

推送成功后，在浏览器打开你的 GitHub 仓库，应能看到最新提交。

## 三、服务器：登录（SSH）

### 3.1 在你自己的 Windows 上打开终端

用 **PowerShell**。若直接输入 `ssh` 报错「找不到命令」，用完整路径：

```powershell
C:\Windows\System32\OpenSSH\ssh.exe ubuntu@你的服务器公网IP
```

- 用户名 **`ubuntu`** 或 **`root`** 以腾讯云控制台说明为准。  
- 密码在控制台「重置密码」里设置。

第一次会问 **`yes/no`**，输入 **`yes`**。

登录成功的标志：出现类似 **`ubuntu@xxxx:~/ $`** 的提示符。

### 3.2 退出 SSH

输入 **`exit`** 回车。

## 四、服务器：更新代码的三种方式

内地服务器访问 **GitHub** 有时会 **SSL 超时**、**git pull 卡住**。下面按 **推荐顺序** 写；**任选一种成功即可**。

### 方式 A：`git pull`（网络正常时最快）

```bash
cd ~/Dota2-Competition
git pull origin main
```

若 **`main`** 报错，试 **`git pull origin master`**，或与你在 GitHub 上实际默认分支名一致。

成功后跳到 **五、重新部署**。

---

### 方式 B：用 WinSCP 从本机覆盖上传（不依赖 GitHub 网络，最稳）

适合：**服务器 git 总失败**，但你本机已经 **`git push` 成功**。

1. 电脑安装 **WinSCP**：**https://winscp.net/**  
2. 新建会话：协议 **SFTP**，主机名填 **公网 IP**，用户名 **`ubuntu`**，密码同 SSH，保存并登录。  
3. 右侧进入服务器目录 **`/home/ubuntu/Dota2-Competition/`**（若用户名不是 ubuntu，路径改成 **`/home/你的用户名/Dota2-Competition/`**）。  
4. 左侧打开你本机项目 **`Dota2-Competition`** 文件夹。  
5. **只改了几个文件时**：把左侧改过的文件拖到右侧 **覆盖**。  
6. **大改版或不确定时**：可以把本机整个项目文件夹打成 zip，上传后在服务器用图形化不好解压时，仍建议在服务器上用 **方式 C** 或保留原 **git clone** 目录只用 WinSCP 同步关键文件。

**重要**：不要删除服务器上的 **`scripts/deploy-aliyun.env`**（里面有你的管理员密码和密钥）。若整目录覆盖前，先在 WinSCP 里把该文件 **备份到桌面再覆盖**，覆盖后再拷回去。

完成后跳到 **五、重新部署**。

---

### 方式 C：用 `wget` 拉 GitHub 源码包（公开仓库时）

仅当仓库为 **Public**。在 SSH 里执行（把 **`Wind297/Dota2-Competition`** 换成你的 `用户名/仓库名`）：

```bash
cd ~
wget --timeout=120 -O /tmp/dota2-main.tar.gz \
  https://github.com/Wind297/Dota2-Competition/archive/refs/heads/main.tar.gz
tar -xzf /tmp/dota2-main.tar.gz
```

解压后目录名一般为 **`Dota2-Competition-main`**。**先备份环境文件**：

```bash
test -f ~/Dota2-Competition/scripts/deploy-aliyun.env \
  && cp ~/Dota2-Competition/scripts/deploy-aliyun.env /tmp/deploy-aliyun.env.bak
```

再替换目录：

```bash
rm -rf ~/Dota2-Competition
mv ~/Dota2-Competition-main ~/Dota2-Competition
test -f /tmp/deploy-aliyun.env.bak \
  && cp /tmp/deploy-aliyun.env.bak ~/Dota2-Competition/scripts/deploy-aliyun.env
```

若 **`mv` 报错找不到目录**，先 **`ls ~`** 看解压出来的文件夹实际名字，把 **`mv` 那一行**里的名字改成一致。

完成后跳到 **五、重新部署**。

## 五、服务器：重新构建并启动（每次代码更新后都做）

脚本名里的 **aliyun** 是历史命名，**腾讯云、任意 Linux + Docker 同样可用**。

### 5.1 你是 `ubuntu` 用户（常见）

在 SSH 里 **一行一行** 执行：

```bash
cd ~/Dota2-Competition
export DATA_HOST_PATH=$HOME/dota2-data
bash scripts/deploy-aliyun.sh
```

- 第一次或依赖变化大时，**构建可能十几分钟**，不要关窗口。  
- 成功结尾会有 **「已启动」** 类提示。

### 5.2 你是 `root` 用户

可以不设 `DATA_HOST_PATH`（脚本默认用 **`/root/dota2-data`**），直接：

```bash
cd ~/Dota2-Competition
bash scripts/deploy-aliyun.sh
```

### 5.3 脚本在做什么（心里有数即可）

- **`docker build`**：按 **`Dockerfile`** 重新打包镜像（含前端构建与 Python 依赖）。  
- **`docker run`**：用 **`scripts/deploy-aliyun.env`** 里的环境变量起新容器；数据库文件在 **`DATABASE_URL=sqlite:////data/competition.db`** 时落在 **`DATA_HOST_PATH` 映射的目录**，一般 **不会因为更新镜像而丢数据**（除非你删了宿主机上那个目录）。

### 5.4 浏览器验证

访问 **`http://你的公网IP:8000/`**（端口以腾讯云防火墙放行为准；若你改过 **`HOST_PORT`**，则改成对应端口）。

## 六、环境变量与登录密码（必读）

- **`scripts/deploy-aliyun.env`**（由 **`deploy-aliyun.example.env`** 复制而来）里的 **`ADMIN_PASSWORD`** 才是 **网站登录密码**。  
- 代码里的默认 **`666`** 只在 **未设置环境变量** 的本地开发时生效；**服务器 Docker 一定会读 env 文件**，所以不要以为线上还是 666。  
- 修改密码：编辑 **`deploy-aliyun.env`** 后 **再执行一遍** **`bash scripts/deploy-aliyun.sh`**。

**不要**把 **`deploy-aliyun.env`**、**PAT**、**真实密码** 发到公开聊天或提交到 Git（本仓库已在 **`.gitignore`** 里忽略 **`scripts/deploy-aliyun.env`**）。

## 八、常见问题

### 8.1 `git push` 要密码不知道填什么

用 GitHub **Personal Access Token** 当作密码；不要用 GitHub 网页登录密码。

### 8.2 服务器 `git pull`：`SSL connection timeout` / `GnuTLS recv error`

内地访问 GitHub 不稳定。**改用 四、方式 B（WinSCP）或 方式 C**。

### 8.3 `docker build` 里 `pip` 超时

仓库内 **`Dockerfile`** 已默认使用 **国内 PyPI 镜像**；若仍超时，多试几次或换网络时段。确认服务器上的 **`Dockerfile`** 已是 GitHub 上最新版（用 WinSCP 覆盖一次也可）。

### 8.4 粘贴密码/token 时出现 `^[[C` 等乱码

不要在密码提示下按 **方向键**；用 **鼠标右键粘贴** 或 **Shift+Insert**，粘贴后 **只按一次 Enter**。

### 8.5 想看清容器是否在跑

```bash
sudo docker ps
sudo docker logs --tail 80 dota2-competition
```

## 九、本地开发（可选，对照用）

仅在你本机调试、不经过 Docker 时：

**后端**（在项目里进入 `backend` 目录，安装依赖后）：

```powershell
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

**前端**（另开一个 PowerShell）：

```powershell
cd frontend
npm install
npm run dev
```

浏览器打开 Vite 提示的地址（多为 **`http://127.0.0.1:5173`**），API 经开发代理转到 8000。默认管理员密码见 **`backend/app/config.py`**（本地未配 env 时多为 **`666`**，与线上一致性无关）。

### 9.1 批量导入初始积分（CSV）

脚本在 **`backend/scripts/import_scores.py`**。在 **`backend` 目录**下执行（路径按你 CSV 实际位置调整）：

```powershell
cd backend
python scripts/import_scores.py ..\initial_scores.csv
```

## 十、最短「发版」检查清单（可收藏）

1. 本机：`git add .` → `git commit -m "..."` → `git push origin main`  
2. 服务器：`ssh` 登录 → **方式 A / B / C** 更新 `~/Dota2-Competition` 代码  
3. 服务器：`cd ~/Dota2-Competition` → `export DATA_HOST_PATH=$HOME/dota2-data`（ubuntu）→ `bash scripts/deploy-aliyun.sh`  
4. 浏览器打开站点试登录

至此完成一次从开发到上线的闭环。
