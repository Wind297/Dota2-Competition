# Dota2 个人积分赛管理后台

面向 **5v5 个人积分赛** 的浏览器管理后台：替代手工记分和筛选，用 **选手池、快捷筛选、比赛创建与结果录入、排行榜与历史** 串起一整场比赛日的日常工作。管理员用 **手机或电脑浏览器** 打开网址即可使用，无需安装客户端。

更早期的设计稿仍保留在仓库内，便于对照：**`产品方案2.md`**（业务规则与数据模型）、**`产品方案3.md`**（在线网站方案与页面设想）。

---

## 1. 产品目标与使用对象

- **目标**：一名比赛管理员在赛前快速筛出可上场选手、创建本场 10 人名单；赛后勾选 5 名胜者，系统自动更新累计积分；随时查看排行榜与历史场次。
- **对象**：熟悉比赛规则、负责排阵与记分的人员（单管理员账号即可）。
- **形态**：单页 Web 应用 + 后端 API，部署后通过 **`http(s)://地址`** 访问；登录后所有数据操作走同一站点。

---

## 2. 管理员典型流程（与界面一致）

1. **登录**  
   打开站点首页，输入管理员密码（由部署时在环境变量中设定，**不是**代码里的演示默认值）。

2. **选手池**  
   - 查看全部选手：**姓名、当前累计积分、今日参赛场数、今日胜场、在线/离线**。  
   - **快捷方案**（如高分场、中分场、补位场等）一键套用筛选；可与 **高级筛选**（积分区间、今日状态、在线状态）组合。  
   - **在线/离线**：可随时切换，用于标记当日不来或临时到场；每个比赛日早间系统会将全员重置为在线（见下文业务规则）。  
   - **勾选恰好 10 人** → **创建比赛**；若违反「同日两场未赛不能占用同一人」等规则，页面会提示错误原因。

3. **比赛详情**  
   - 创建成功后进入该场详情，展示 10 名上场选手。  
   - 赛后勾选 **恰好 5 名胜者** → **提交结果**：积分按胜场 +1，比赛记为已完成。  
   - **已完成的比赛** 仍可在详情中 **更正胜者名单**（系统按差额调整积分）；也可 **删除比赛**（若已计分会先回退胜场加分，再删除记录）。

4. **历史比赛与排行榜**  
   - **历史比赛**：按状态、比赛日等查看列表，进入单场详情。  
   - **排行榜**：按累计积分降序排列。

5. **选手维护（按需）**  
   - 新增单个选手、**批量导入**（CSV）、编辑信息；若选手已进入某场比赛关联，删除会受限（避免数据不一致）。

## 3. 核心业务规则（摘要）

以下内容与当前实现一致；档位与「今日」统计均基于 **比赛日** 与 **Asia/Shanghai** 时区逻辑。

### 3.1 比赛日

- 一个比赛日从 **当日 09:00:00** 起，至 **次日 08:59:59** 止。  
- 每场比赛归属其 **创建时刻** 所在的比赛日。

### 3.2 积分与档位

- **积分**：整数，等于 **累计胜场**（每胜一场 +1）。  
- **档位**（左闭右开，用于筛选与快捷方案）：  
  - 低分档：0～4  
  - 中分档：5～9  
  - 高分档：≥10  

### 3.3「今日」相关状态（用于筛选）

- **今日未参赛**：当日参赛场数为 0。  
- **今日未获胜**：当日胜场数为 0。  
- **今日参赛 ≤1 场**：当日参赛场数 ≤1。  
- 其他组合见界面上的「今日状态」筛选项。

### 3.4 在线状态

- 进入 **新的比赛日** 后，当管理员 **正常使用后台**（有请求进入服务端）时，系统会将全员 **在线状态重置为「在线」**（与比赛日切换逻辑配合，而非依赖本机定时任务）。  
- 管理员可随时把某人标为离线或恢复在线，便于现场沟通。

### 3.5 快捷筛选方案（与档位、今日、在线组合）

与实现一致的方案名称包括（具体条件以接口 **`GET /api/presets`** 与界面为准）：**高分场、中分场、低分场、补位场、红利场、加赛场、附加赛场** 等，用于一键缩小选人范围。

### 3.6 比赛人数与并发

- 创建比赛必须 **恰好 10 名不同选手**。  
- 提交结果必须 **恰好 5 名胜者**，且均在本场 10 人名单中。  
- **同一比赛日内**：同一选手 **不能同时出现在两场「已确认且未完成」** 的比赛中（避免一人被排进两场未赛）。

### 3.7 比赛状态

- **已确认**：名单已定，尚未提交结果。  
- **已完成**：已提交结果（或后续在已完成状态下修正过胜者）。  
- 历史方案中曾描述「不支持取消」；当前版本在管理需要的前提下支持 **删除比赛**（会回退已产生的胜场加分）及 **已完成场次的胜者更正**。

## 4. 技术实现概要（非操作手册）

| 层级 | 说明 |
|------|------|
| 后端 | Python **FastAPI**，REST API 前缀 **`/api`**；登录 **`POST /api/auth/login`**，后续请求携带 Bearer Token。 |
| 前端 | **Vue 3 + Vite + Naive UI**，构建后静态资源由同一后端进程挂载，单服务部署。 |
| 数据 | 默认 **SQLite**；生产可切换 **PostgreSQL**（`DATABASE_URL`）。比赛日、时间展示等见后端 `matchday`、`datetime_norm` 等模块。 |
| 交付 | 根目录 **`Dockerfile`**：镜像内构建前端并复制到 `backend/static`，进程为 **uvicorn**。 |

初始积分可通过脚本从 CSV 导入（列 **`name`**, **`score`**），脚本路径为 **`backend/scripts/import_scores.py`**（在 `backend` 目录下按该脚本内说明或原 README 习惯执行即可）。
